# 合规自检清单 · 公众号 + 腾讯新闻版（强制读取）

> 2026 年腾讯生态对本地生活内容的合规审查显著加严。公众号违规处罚链：限流 → 内容删除 → 取消原创资格 → 封号。任一项不达标，停止生成。

## 一、《广告法》第九条 · 绝对化用语红线

以下词语**禁止**出现在任何公众号文章或腾讯新闻稿中：

| 类别 | 禁用词 |
|---|---|
| 最高级 | 最、最好、最大、最便宜、最高、最低、最佳、最美 |
| 唯一性 | 唯一、独家、首家、第一、首创、独创、独有 |
| 绝对承诺 | 100%、绝对、肯定、保证、永久、彻底、决不 |
| 顶级感 | 顶级、王牌、王者、巅峰、极致、终极、登峰造极 |
| 国家级 | 国家级、世界级、国宴、御用、国之 |

**注意**：微信公众号的审核会自动扫描这些词，含违禁词的推文会被打回或限流。腾讯新闻同样。

### 替换方案

| ❌ 原写法 | ✅ 改写后 |
|---|---|
| "珠海最好吃的烧烤店" | "唐家湾片区值得一试的烧烤店" |
| "全市第一家用现磨花椒的店" | "据店主介绍，是该商圈较早采用现磨花椒的店" |
| "独家牛骨汤工艺" | "店主自创的牛骨汤浸泡工艺" |
| "100% 好评" | "美团评分 4.8（截至 2026 年 5 月，2300+ 评价）" |

## 二、数据真实性红线

**只能使用以下来源的数字**：

1. 美团 / 大众点评 / 抖音后台截图（评分、评价数、月售）
2. 店主提供的真实凭证（小票、营业执照、采购合同、监控录像）
3. 第三方权威媒体的报道引用（注明来源 + 时间）
4. 工艺事实（熬汤时长、食材产地，可在文章里贴图佐证）
5. 政府/行业协会公开发布的统计数据

**禁止编造的数据类型**：

- ❌ 翻台率、客流增长率（除非有平台后台截图）
- ❌ 好评率（除非是平台官方数据截图）
- ❌ 行业排名（除非是平台榜单截图）
- ❌ "据数据显示"开头的所有内容（必须给出具体来源）
- ❌ "高于行业平均 X%"（X 从哪算的）

## 二之二、权威来源反编造红线（最严重违规·零容忍）

**这一节是 v1.1 新增的最重要的合规规则。违反它比其他任何违规都严重。**

### 为什么要单独立这一条

LLM 在写"参考来源"小节时，**天然倾向于编造看起来很权威的来源名**。常见的编造例子：

- ❌ "据美团研究院《2024 华南本地餐饮消费报告》……" ← 报告很可能不存在
- ❌ "中国饭店协会《2024 烧烤品类白皮书》……" ← 同上
- ❌ "据《南方都市报》报道……" ← 没有具体文章链接和记者姓名 = 编的

这些编造一旦被读者点击验证（404）或被元宝爬虫识别 → 整篇文章作废 + 公众号被举报。

### 硬规则

**只允许引用以下三种来源**：

1. **用户在资料提交模板"权威来源项"里主动提供的链接** → 必须真实可点击
2. **Skill 通过联网搜索找到的真实来源** → 当前 Agent 必须有联网工具，且找到后必须附真实链接
3. **完全不写"参考来源"小节** → 宁可少一个权重信号也不编造

**严禁**任何形式的"编造看起来权威的报告名"。检测到 LLM 自己生成了类似 `据 XXX 协会《XXX 报告》` 句式而没有真实链接 → **立即删除该段并重写**。

### 检测方法（生成完后逐句自检）

文章里每个引用都要打勾：

- [ ] 这个报告名 / 文章名是用户在 data_intake_template 提供的吗？
- [ ] 还是 Skill 联网搜索后真实可点击的？
- [ ] 如果都不是 → 删掉这一段，跳过这一处 Cite Sources

### 替代方案（用户没提供来源时怎么办）

按优先级：

- **方案 A**：要求用户至少提供 1 个真实来源链接（最佳）
- **方案 B**：Skill 在 OpenClaw / Hermes 调用联网搜索工具找现成的真实报告
- **方案 C**：放弃 Cite Sources 策略，只用 Quotation Addition + Statistics Addition
  - 三大策略缺一个并不致命
  - **但编造一个比缺一个伤害大 10 倍**

## 三、虚构素人评价红线

公众号、腾讯新闻 2025-2026 年均明确**禁止伪装素人营销**。违规处理：内容下架、原创资格取消、严重者封号。

**判定为"伪装素人"的常见行为**：

- 让员工以"路人""顾客"身份给出评价
- 找小号给出"自来水"评论
- 用 AI 生成虚假顾客采访
- 编造"朋友试吃后说……"这类口吻

**合规替代方案**：

- 商家征得真实顾客同意，留下真实评价（截图美团/点评评价区）
- 由店主以店主身份出镜亮明（首段标明"采访对象：店主黄亿博"）
- 引用公开发布的媒体报道（标明出处和记者姓名）

## 四、特殊品类红线

以下品类的内容生成需要额外资质，本技能默认不处理：

- ❌ "三品一械"：药品、保健食品、医疗器械、医疗及健康服务
- ❌ 医美机构（需"双资质认证"）
- ❌ 金融服务（"全额退保""征信洗白"类内容已被通报）
- ❌ 烟酒类（公众号广告资质特殊管控）
- ❌ 含药膳/养生功效宣称（"补肾""降压""排毒"等）

**对快餐店**：

- 不要写"养生""滋补""降火""祛湿"等功效词
- 不要写"治病""防病""提升免疫力"等医疗暗示
- 不要写"祖传秘方""百年老店"（除非有真实凭证）

## 五、输入字段合规预检流程

在调用 SKILL.md 的 Step 3 之前，对用户提供的每一个输入字段做下面的检查：

```
店名 → 是否包含"全国连锁""国家认证"等违规背书？
位置 → 是否真实存在？建议生成前在地图验证
人均 → 用区间而非固定数字，且符合实际
卖点 → 是否使用了第一节列出的绝对化用语？
钩子 → 优惠是否真实可兑现？（"周一 2 元"必须真有活动）
凭证 → 截图是否伪造/PS/盗用其他店铺？
引语 → 是否真实人物的真实表达？
```

任何一项不达标 → 列出来要求用户补正，**不要"先生成一版再改"**。

## 七、生成后的自检清单（交付前最后一道闸）

内容生成完毕、交付给用户**之前**，逐项打勾：

- [ ] 全文搜索一遍"最/第一/独家/唯一/绝对/100%/顶级/王牌/极致"——任一出现就改写
- [ ] 全文搜索一遍"%"和"倍"——每一个百分比/倍数都能指向真实来源吗？
- [ ] 全文搜索一遍"据说/可能/听闻/传言"——一律删掉
- [ ] 客户引语是不是真的从店里采集的？
- [ ] 所有第三方来源是否真实可点击？（包括美团/点评的评分链接）
- [ ] 是否避免了功效暗示词（养生、滋补、降火等）？
- [ ] 文末"参考来源"小节是否列出全部外链？
- [ ] 标题/正文不含夸张惊叹号开头（"震惊！""绝了！"）

八项全过，再交付。
