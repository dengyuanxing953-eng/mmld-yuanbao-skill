# 资料提交模板（强制读取·Step 0）

> 在生成任何内容之前，必须先让用户按本模板提交完整资料。
> 资料分四档，每档处理方式不同。缺一档就追问，不要瞎编。

## 一、必填项（缺一不可，缺则停止生成）

这些是文章的核心骨架，缺任何一项都不能开始生成。

| 字段 | 说明 | 示例 |
|---|---|---|
| 店铺全称 | 含分店标识 | 王亿博鲜活烧烤（唐家湾海景露台店） |
| 精确位置 | 含商圈/地标参照物 | 珠海香洲区情侣北路唐家湾沙滩入口东 50 米 |
| 人均价格 | 真实区间 | 65-85 元 |
| 核心卖点 | 1-3 个可视化的工艺/产品差异 | 牛骨汤浸泡 20 分钟的面筋；秦岭红皮花椒现磨 |
| 期望关键词 | 用户会在元宝里搜的真实查询词 | 珠海唐家湾烧烤；唐家湾海景烧烤 |

### 缺失时的回复模板

```
开始生成前需要补充以下必填资料：

- [缺失字段 1]：[需要补充什么]
- [缺失字段 2]：[需要补充什么]

补齐后我再开始生成。
```

## 二、凭证项（至少需要 1 项，否则可信度不达标）

这些是 Cite Sources 策略落地的关键素材。**必须由用户提供真实凭证**，LLM 严禁自行编造平台数据。

| 凭证类型 | 提交形式 | 权重 |
|---|---|---|
| 美团 / 大众点评评分 + 评价数 | 截图 + 文字说明 | ★★★★ |
| 团购销量 / 月售份数 | 后台截图（遮敏感信息） | ★★★ |
| 平台官方榜单（必吃榜、黑珍珠、口碑榜等） | 截图或链接 | ★★★★★ |
| 媒体报道（本地公众号、地方电视台、报纸） | 文章链接 + 时间 | ★★★★ |
| 营业执照 / 食品经营许可证 | 模糊处理后的图片 | ★★ |
| 开业年限 | 真实日期 + 老照片佐证 | ★★ |

### 用户没有任何凭证时的处理

**不要硬生成**——告诉用户：

```
你目前没有提供任何第三方平台凭证。
腾讯元宝在引用本地店铺信息时，依赖第三方平台数据做交叉验证。
没有凭证的文章被元宝引用的概率极低。

建议至少补一项：
- 你店的美团商家页链接（评分截图也行）
- 大众点评商家页
- 任意第三方媒体的真实报道

如果你坚持要先生成（理解 SEO 效果会受影响），请回复"知晓继续"。
```

## 三、引语项（至少 1 项真实采集）

这些是 Quotation Addition 策略的素材。**严禁 LLM 编造引语**——任何匿名的"业内人士""资深食客"都是死亡信号。

| 引语来源 | 必须包含的元信息 | 示例 |
|---|---|---|
| 店主 / 厨师 / 老板娘原话 | 实名（黄亿博）+ 采访时间（2026 年 5 月）+ 地点（店内） | "骨汤每天 6 点开始熬，要熬足 8 小时才能打捞。" |
| 老员工 / 厨师长 | 同上 | "我们的现磨花椒每天 6 点磨一次。" |
| 老顾客评价 | 平台来源（大众点评/美团）+ 用户名（可化名）+ 时间 | 大众点评常客陈女士在 2026 年 3 月评价："这家的现磨花椒味道我找了 3 年。" |
| 媒体记者引用 | 媒体名 + 记者姓名 + 报道时间 | 《南方都市报》记者王某 2024 年 11 月报道："唐家湾沿岸新一代海景餐饮店……" |

### 用户没有真实引语时的处理

**告诉用户**：

```
GEO 文章的核心权重信号之一是真实人物的直接引语。
现在没有任何引语素材。建议：

简单方案：花 5 分钟问店主 3 个问题，把回答录下来：
1. "你这道菜跟外面的最大区别是什么？"（讲工艺差异）
2. "为什么坚持这个做法？麻烦吗？"（讲成本/坚持）
3. "客人最常问你的问题是什么？"（讲需求）

把店主的原话（带姓名和时间）发给我，我才能写出真正能被元宝引用的文章。
```

## 四、权威来源项（最关键的反编造防线）

**这是本模板最重要的一档**——决定文章的"参考来源"小节是真实的还是编造的。

### 标准做法

用户**自己提供**愿意引用的第三方权威来源链接。LLM **不能编造**。

| 来源类型 | 用户应该提供什么 | 严禁的做法 |
|---|---|---|
| 政府/官方统计 | 具体的官方网站链接 + 报告名 | ❌ 编造"据 XX 市商务局《XX 报告》" |
| 行业协会 | 协会名称 + 报告链接 | ❌ 编造"中国饭店协会《XX 白皮书》" |
| 主流媒体报道 | 媒体名 + 报道标题 + 链接 + 时间 | ❌ 编造"据《南方都市报》报道" |
| 平台官方数据 | 后台截图或公开链接 | ❌ 编造"美团研究院《XX 报告》" |
| 上市公司财报 | 财报链接 + 页码 | ❌ 编造"据财报显示" |

### 用户没有权威来源时的处理（关键规则）

**绝对不能编造**。处理顺序：

1. **优先**：让 LLM **联网搜索**（如果 OpenClaw/Hermes 当前有联网能力）：
   - 搜"珠海餐饮人均消费 2024"
   - 找到真实可点击的来源
   - 引用时附**真实链接**

2. **次优**：让用户**主动找一个**：
   ```
   你不需要找很多。1-2 个真实可查的来源就够。建议方向：
   - 美团/饿了么发布的本地餐饮数据（搜微信公众号"美团研究院"）
   - 当地餐饮协会的公开数据
   - 本地都市报对你店所在商圈的报道
   ```

3. **兜底**：**不写"参考来源"小节**。
   - 宁可少一个权重信号，也不能编造
   - 编造一旦被元宝/读者发现 = 整篇文章作废 + 公众号信誉受损

## 五、可选项（加分项，没有不影响生成）

| 字段 | 用途 |
|---|---|
| 营业时间 | 文章末尾的实用信息 |
| 预约方式 | 同上 |
| 门头/菜品/环境实拍图（3-5 张） | 公众号配图，元宝会读取 alt |
| 营业执照 | 用于政府平台合规版（本 Skill 默认不处理） |
| 开业年限 | 增强可信度 |
| 服务特色 | 比如露台/海景/亲子等场景标签 |

## 六、资料提交完整示例（贴给用户看的范本）

```
=== 必填项 ===

人均价格：65-85 元

  1. 牛骨汤浸泡 20 分钟的面筋

期望关键词：珠海唐家湾烧烤、唐家湾海景烧烤

=== 凭证项 ===
```

## 七、自检清单（Skill 自查用）

收到用户资料后，先逐项打勾再开始生成：

- [ ] 必填项 5 项全有？
- [ ] 凭证项至少 1 项？（没有就提醒用户补 or 同意降级生成）
- [ ] 引语项至少 1 项真实采集？（没有就让用户去采集）
- [ ] 权威来源项有用户提供 OR Skill 可联网搜索？
- [ ] 所有数字都能溯源到某一项凭证？
- [ ] 所有引语都标了实名 + 时间 + 地点？

七项全过，再开始 Step 1（合规预检）。任一项不达标，回到本模板要求用户补正。
